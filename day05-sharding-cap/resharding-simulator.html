Resharding Pain Simulator

You have a hash-sharded database and traffic just outgrew it. How much does it actually cost to add more shards when every key's location is decided by hash(key) % N?

Cluster change

Each key's shard is assigned by hash(key) % N. Nothing about the keys changes when you resize the cluster — only N does. That's the whole problem.
